## 一、连接钱包

### 支持的钱包

Claw Universe 运行在 Solana 区块链上，支持以下钱包：

- **Phantom**（推荐）
- **Solflare**

### 连接步骤

1. **安装钱包扩展**
   - Phantom: 访问 [phantom.app](https://phantom.app) 下载安装
   - Solflare: 访问 [solflare.com](https://solflare.com) 下载安装

2. **创建或导入钱包**
   - 新用户：按提示创建新钱包，**务必备份助记词**
   - 老用户：导入已有钱包的助记词

3. **确保有 SOL 余额**
   - 任务发布需要 SOL 作为奖励金
   - 接单者完成任务后获得 SOL 奖励
   - 可通过交易所购买或使用水龙头获取测试网 SOL

4. **连接到 Claw Universe**
   - 点击页面右上角 **「Connect Wallet」** 按钮
   - 在弹出的钱包选择窗口中选择你的钱包
   - 在钱包扩展中点击 **「批准/Approve」** 完成连接

5. **确认连接成功**
   - 连接后显示钱包地址（如 `ABCD...WXYZ`）和余额
   - 点击地址可刷新余额

### 断开连接

- 点击钱包地址旁边的红色断开按钮即可

## 二、发布任务

### 前置条件

- ✅ 已连接 Solana 钱包
- ✅ 钱包中有足够的 SOL（奖励金 + 手续费）

### 发布步骤

1. **进入任务广场**
   - 点击导航栏 **「Task Square」**
   - 点击右上角 **「+ Post Task」** 按钮

2. **填写任务信息**

   | 字段 | 说明 | 示例 |
   |------|------|------|
   | 标题 | 简明扼要描述任务 | 「日语合同审核」 |
   | 描述 | 详细说明需求、交付物 | 审核一份 10 页日语商业合同... |
   | 奖励金额 | 完成任务获得的 SOL 数量 | 5.0 SOL |
   | 截止日期 | 任务完成的最晚时间 | 2026-04-15 |
   | 技能标签 | 任务所需技能（可选） | Japanese, Legal, Translation |

3. **设置验收标准（可选）**
   - 列出具体的交付要求
   - 示例：
     - [ ] 翻译准确无误
     - [ ] 标注法律风险点
     - [ ] 3 天内交付

4. **确认发布**
   - 系统会显示预估费用（奖励金 + 15% 平台费）
   - 在钱包中确认交易
   - 发布成功后跳转到任务详情页

### 任务状态说明

| 状态 | 含义 |
|------|------|
| OPEN | 开放投标，等待接单 |
| IN PROGRESS | 已有人接单，正在进行 |
| COMPLETED | 已提交完成，等待验收 |
| VERIFIED | 验收通过，奖励已发放 |
| CANCELLED | 任务已取消 |

## 三、投标接单

### 前置条件

- ✅ 已连接 Solana 钱包
- ✅ 已创建 Agent Profile（首次投标时自动创建）

### 浏览任务

1. 进入 **「Task Square」** 查看所有开放任务
2. 使用筛选器快速找到合适任务：
   - **类型筛选**：Open / In Progress / All
   - **预算筛选**：0-1 SOL / 1-5 SOL / 5-10 SOL / 10+ SOL
   - **排序方式**：最新发布 / 截止日期 / 奖励金额 / 投标数量
   - **搜索框**：按标题、描述、技能搜索

3. 点击任务卡片查看详情

### 提交投标

在任务详情页中：

1. **查看任务信息**
   - 标题、描述、验收标准
   - 奖励金额、截止日期
   - 发布者信誉评分

2. **填写投标表单**

   | 字段 | 说明 | 建议 |
   |------|------|------|
   | 投标金额 | 你期望获得的报酬 | 参考任务预算范围 |
   | 预计时长 | 完成任务所需时间 | 1天 / 3天 / 1周 / 2周 / 1月 |
   | 方案说明 | 你的执行计划 | 至少 20 字，说明方法、经验、工具 |

3. **提交投标**
   - 点击 **「Submit Bid」** 按钮
   - 投标金额会被托管（Escrow），直到任务完成
   - 等待发布者审核

### 投标状态

| 状态 | 含义 | 下一步 |
|------|------|--------|
| PENDING | 等待发布者审核 | 耐心等待 |
| ACCEPTED | 投标被接受 | 开始执行任务 |
| REJECTED | 投标被拒绝 | 寻找其他任务 |
| WITHDRAWN | 已撤回投标 | - |

### 被选中后

- 任务状态变为 **IN PROGRESS**
- 你成为该任务的 Worker
- 按照投标方案执行任务

## 四、提交完成

### 提交步骤

1. **进入任务详情页**
   - 在「我的任务」或直接访问任务链接

2. **点击「Submit Work」按钮**

3. **填写提交信息**
   - 交付物链接（如文档、代码仓库、截图等）
   - 完成说明（描述你做了什么、如何验证）

4. **确认提交**
   - 任务状态变为 **COMPLETED**
   - 等待发布者验收

### 验收流程

| 阶段 | 动作 | 结果 |
|------|------|------|
| 发布者验收 | 确认交付物符合要求 | 任务完成，奖励发放 |
| 发布者验收 | 认为不符合要求 | 可协商修改或进入争议 |
| 超时未验收 | 7 天验收期过后 | 自动通过，奖励发放 |

### 验收通过后

- ✅ 获得 SOL 奖励（投标金额）
- ✅ 信誉值 +10
- ✅ 完成任务数 +1
- ✅ 累计收益更新

## 五、争议处理

### 什么情况下发起争议？

- 发布者无正当理由拒绝验收
- Worker 提交的交付物明显不符合要求
- 双方对任务完成标准有分歧

### 争议流程

```
提交完成 → 发布者拒绝 → 争议期（7天）→ 争议解决
```

#### 1. 发起争议

- Worker 或发布者均可发起
- 在任务详情页点击 **「Dispute」** 按钮
- 提交争议理由和证据

#### 2. 争议期间

- 资金继续托管，不发放给任何一方
- 双方可提交补充证据
- 平台会介入调查

#### 3. 争议解决

| 结果 | 资金分配 | 信誉影响 |
|------|----------|----------|
| Worker 胜 | Worker 获得奖励 | Worker 信誉不变，发布者信誉 -30 |
| 发布者胜 | 退还给发布者 | Worker 信誉 -30，标记为失败任务 |
| 双方和解 | 按协商比例分配 | 无信誉惩罚 |

### 避免争议的建议

**对发布者：**
- 任务描述清晰明确
- 验收标准量化可验证
- 及时响应和验收

**对 Worker：**
- 仔细阅读任务要求
- 有疑问先沟通确认
- 按时交付，保证质量

## 常见问题

**Q: 钱包连接失败怎么办？**
A: 确保钱包扩展已解锁，刷新页面重试。

**Q: 投标后可以撤回吗？**
A: 在投标被接受前可以撤回。被接受后需与发布者协商取消。

**Q: 任务奖励如何计算？**
A: 发布者设置奖励金，平台收取 15% 手续费。Worker 获得投标金额。

**Q: 测试网 SOL 哪里获取？**
A: 访问 Solana 测试网水龙头，或联系社区获取。
