## 一、KYC 等级设计

### 1.1 等级总览

```
┌─────────────────────────────────────────────────────────────────┐
│                     Claw Universe KYC 体系                       │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│   Lv0 匿名层         Lv1 验证层          Lv2 实名层              │
│   (钱包直达)         (邮箱核验)         (身份核验)               │
│                                                                  │
│   ├── 无需注册        ├── 邮箱验证         ├── 身份证/KYC服务商    │
│   ├── 纯钱包地址      ├── 基本访问         ├── 高级功能解锁         │
│   └── 功能受限        └── 任务浏览         └── 完整市场参与        │
│                                                                  │
│   ⚠️ AML 基础监控始终开启（链上行为分析）                        │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

### 1.2 KYC 等级对比表

| 维度 | Lv0 匿名 | Lv1 邮箱验证 | Lv2 身份验证 |
|------|----------|--------------|--------------|
| **标识符** | 钱包地址 | 邮箱 + 钱包地址 | 身份证/KYC Token + 钱包地址 |
| **验证方式** | 无 | 邮箱验证码（6位 OTP） | KYC 服务商（自动+人工审核） |
| **数据收集** | 无（仅链上地址） | 邮箱地址 | 姓名+身份证+手机号+人脸（依据 JAEPA） |
| **可执行操作** | 浏览公开任务 | 浏览 + 接单 + 发布低额任务（≤500 CLAW） | 全功能解锁 |
| **任务限额** | 仅浏览 | 单笔 ≤500 CLAW，累计 ≤5,000 CLAW/月 | 无上限 |
| **提现限额** | 无 | 单笔 ≤1,000 CLAW | 无 |
| **适用场景** | 单纯浏览 | 小额任务、测试网络 | 正式商业行为、大额交易 |
| **合规依据** | FATF Travel Rule（可选） | 日本资金决済法（小声买卖取引）| 犯罪收益移転防止法（犯収法）|
| **数据保留期** | 链上日志永久 | 5年（日本税务合规） | 7年（犯収法要求） |
| **KYC 成本** | $0 | $0（自托管 OTP） | $1~3/人（第三方） |
| **风险等级** | 🔴 高（无摩擦 AML） | 🟡 中（基础可追溯） | 🟢 低（完整身份链） |

### 1.3 升级路径设计

```
钱包连接
    │
    ▼
[Lv0 匿名] ──── 领取新手任务（≤100 CLAW）────► 引导验证邮箱
    │                                              │
    │                                         [Lv1 邮箱]
    │                                              │
    │                         发布任务 ≥500 CLAW / 提现 ≥1000 CLAW
    │                                              │
    └──────────────────────► 引导完成身份验证 ◄────┘
                               │
                          [Lv2 实名]
                               │
                         全功能解锁
```

**设计原则**：
- 渐进式验证：不在入口拦截，降低用户流失
- 功能驱动升级：以业务需求触发验证，而非强制注册
- 数据最小化：Lv0 不收集任何个人数据

## 二、合规考量

### 2.1 日本金融厅（FSA）监管框架

```
┌─────────────────────────────────────────────────────────────────┐
│              日本加密资产合规框架（相关法律）                    │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  法律                          适用场景                          │
│  ──────────────────────────────────────────────────────────     │
│  犯罪收益移転防止法（犯収法）  加密资产交换业（JVCEA 注册）      │
│  资金决済法（資金決済法）      虚拟通货交換業（暗号資産交換業）  │
│  金融商品取引法（金融商品取引法） token 视为「暗号資産」时的规制  │
│  个人信息保护法（PIPA）        所有个人数据处理                   │
│  外国規制强化への対応           跨境用户识别                     │
│                                                                  │
│  ⚠️ 若 CLAW Token 被认定为「暗号資産」（加密资产）：              │
│     → 必须完成 JVCEA（、日本加密资产交換业协会）注册             │
│     → 适用犯収法的交易时身份确认义务                              │
│     → 交易报告义务（可疑交易申报）                               │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

#### 犯収法核心要求（对 Claw Universe 的影响）

| 要求 | 具体内容 | Claw Universe 实现方案 |
|------|----------|----------------------|
| **客户身份确认（CAP）** | 交易时确认客户姓名+地址+出生年月日 | Lv2 强制完成，否则无法进行「暗号資産」交易 |
| **记录保存** | 确认记录保存7年 | 所有 KYC 数据（加密存储）保留7年 |
| **可疑交易申报（STR）** | 发现可疑交易时向 FIA 申报 | AML 监控模块，对接链上分析工具 |
| **黑名单筛查** | 与制裁名单（OFAC/FATF）比对 | Lv1+ 用户地址实时筛查 |

### 2.2 数据隐私保护（PIPA + GDPR 基准）

```
┌─────────────────────────────────────────────────────────────────┐
│                    数据处理合规矩阵                              │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  数据类别              处理目的              法律依据            │
│  ─────────────────────────────────────────────────────────     │
│  钱包地址              链上身份标识          合法利益（防欺诈）    │
│  邮箱地址              账户验证/通知         用户同意             │
│  姓名+身份证号          身份确认（JAEPA）    法律义务（犯収法）   │
│  手机号               二次验证/通知          用户同意             │
│  人脸生物特征          Lv2 补充验证          明确同意+加密存储    │
│  链上交易记录          AML监控/税务          合法利益（合规）     │
│                                                                  │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │  隐私保护设计（Privacy by Design）：                      │   │
│  │  1. 数据最小化：Lv0 不收集任何 PII                       │   │
│  │  2. 目的限制：KYC 数据仅用于合规目的，不可用于营销       │   │
│  │  3. 加密存储：所有 PII 使用 AES-256 静态加密            │   │
│  │  4. 访问隔离：KYC 数据与业务数据物理分离                 │   │
│  │  5. 数据销毁：用户注销后按保留期删除（可配置）            │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

### 3.3 成本估算（MVP 阶段）

| Lv2 身份验证 | Sumsub | - | $1.5/次 | 按需付费 |

| **合计（MVP）** | | **~$350/月** | **按量付费** | |

> ⚠️ **犯収法合规临界点**：当平台月交易额 ≥ ¥200万日元 或 月活跃用户 ≥ 100人时，必须完成 JVCEA 注册并满足犯収法全部要求。建议 MVP 阶段控制在门槛以下，或提前完成注册。

### 3.4 集成实施路线图

```
Phase 1（MVP，0→1）：
├── 集成 Merkle Science（AML 监控）

Phase 3（规模化，1000+用户）：
├── 申请 JVCEA 注册（暗号資産交換業登録）
├── Chainalysis 升级（高级调查工具）
└── 聘请专职合规负责人（ bileye 登备）
```
