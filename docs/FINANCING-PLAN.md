## 五、风险对冲策略

### 5.1 融资失败应对预案

```
【Pre-Seed融资失败（4月）】
触发条件：4月30日前未完成¥200,000最低金额
应对策略：
  1. 创始团队降薪50%维持运营（节省¥20,000/月）
  2. 削减基础设施成本（只用免费层）
  3. 法务合规延后（先跑通产品，技术债务）
  4. 寻找YC中国/科技寺等孵化器（可获¥100,000+支持）
  5. 若6月底仍无资金：暂停项目或Pivot

【Seed融资失败（7月）】
触发条件：7月15日前未完成¥1,000,000最低金额
应对策略：
  1. 裁员至2人核心团队（节省¥80,000/月）
  2. 无限期延后合规（仅面向全球用户运营）
  3. 寻找被收购机会（已有Demo可谈判）
  4. 申请政府创业补贴（硅谷/东京都有）

【Series A融资失败（2027Q1）】
触发条件：2027年Q1未完成¥3,000,000
应对策略：
  1. 靠平台收入自给自足（月GMV × 15%抽成）
  2. 削减营销预算，聚焦产品
  3. 考虑Bridge Round（过桥融资）
  4. 放缓扩张速度
```

### 5.2 预算削减优先级

```
当现金流紧张时，按以下顺序削减：

第1刀（-¥50,000/月）：暂停付费广告投放
第2刀（-¥20,000/月）：削减KOL合作
```

## 七、总结

### 7.1 核心数字

```
Year 1 融资目标：¥7.0M - 9.5M
  Pre-Seed：¥400,000（4月）
  Seed：¥2,000,000（7月）
  Series A：¥7,000,000（2027Q1）

Year 1 支出预算：¥6.0M - 9.8M（保守-乐观）
  保守：¥6.0M（核心功能，最小团队）
  中间：¥7.5M（计划内）
  乐观：¥9.8M（规模扩张）

现金流红线：
  ⚠️ 6月底前Pre-Seed必须到账
  ⚠️ 9月底前Seed必须到账
  ⚠️ 2027年Q1前Series A必须到账
```

### 7.2 关键成功因素

```
1. 【速度】融资要在钱花完之前3个月启动，不能等到弹尽粮绝
2. 【门槛】每个融资轮次必须有硬性里程碑，不能凭PPT融资
3. 【估值】估值要合理，早期估值过高会伤害后续融资
4. 【储备】永远留6个月运营资金作为安全垫
5. 【止血】发现资金不够时，第一反应是削减成本而不是找钱
```
