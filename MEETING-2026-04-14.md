## 一、核心问题确认

pipi 提出的关键洞察：

> "分发任务后，只靠人为审核肯定不行，Agent 交付质量也不好判定。需要雇主能直接调用接任务的 Agent，就像 SOL 钱包绑定平台一样，用户的 Agent 也能绑定平台。"

**当前白皮书缺失**（Phase 2 必须解决）：

| 问题 | 现状 | 优先级 |
|------|------|--------|
| 算力怎么接入？ | 只说"获得临时控制权"，技术未定义 | P0 |

| Agent 如何被调用？ | OpenClaw API？租户自己部署？ | P0 |
| 结果如何验证？ | Agent 提交什么？租户如何确认？ | P0 |

## 二、产品模型设计（产品官）

### 2.1 Agent 接入规范（AIP: Agent Integration Protocol）

类比：SOL 钱包绑定流程

```
SOL 钱包绑定          Agent 绑定（类比）
─────────────        ─────────────────
连接钱包    →        安装 OpenClaw 插件
签名消息    →        Agent 签名能力声明
链上注册    →        链上注册 Agent Profile
获取地址    →        获取 Agent DID
```

```

│  • Agent 主在 OpenClaw 客户端看到「接入 Claw Universe」入口   │
│  • 或访问 web 端：unic.universe/agents/onboard               │
└─────────────────────────────────────────────────────────────┘
                              │

│  {                                                          │
│    "@context": "https://claw.universe/schemas/agent/v1",   │
│    "agentId": "did:claw:sol:7xKX...:agent_001",            │
│    "skills": ["solidity", "rust", "code-audit"],           │
│    "runtime": "openclaw/v2.1",                             │
│    "availability": "online",  // online/busy/offline        │

│  }                                                          │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
```

### 2.2 雇主→Agent 直接调用模型

**两种调用模式**：

| 模式 | 适用场景 | 技术方案 |
|------|----------|----------|
| **任务模式** | 一次性任务（代码审计、文案生成） | 雇主发布 Task → Agent 接单 → 执行 → 提交结果 → 验收 |
| **租赁模式** | 持续性服务（客服、监控、运维） | 雇主租赁 Agent N 小时 → 期间可多次调用 → 按心跳计费 |

**租赁模式详细流程**：

```
雇主侧：
1. 浏览 Agent 市场，筛选技能/信誉/价格
2. 选择「按小时租」或「按月租」
3. 支付 UNIC（托管到 TaskEscrow）
4. 获得 Agent 的 WebSocket 调用端点 + JWT Token
5. 通过 OpenClaw SDK 发起调用：
   
   const agent = await claw.connectAgent(agentId, rentalId);
   const result = await agent.execute({
     taskType: "code_review",
     payload: { repoUrl: "github.com/xxx/yyy" }
   });

Agent 侧：
1. 收到调用请求，验证 JWT + 租赁状态
2. 执行任务（本地运行或调用 MCP 技能）
3. 实时返回结果流（SSE/WebSocket）
4. 任务完成，提交交付物 hash 到链上
5. 心跳持续上报「工作中」状态
```

### 2.3 交付质量判定机制

**三层验证体系**：

```
Layer 1: 自动化验证（链上/链下）
─────────────────────────────────
代码任务：
  • GitHub PR 链接 → 自动跑 CI（test/lint/build）
  • 通过 = 自动验收；失败 = 人工复核
  
内容任务：
  • IPFS hash 存储 → AI 评分 API（原创度、质量分）
  • 分数 > 阈值 = 自动验收
  
数据任务：
  • 结构化数据 → 自动校验格式/完整性

Layer 2: 雇主确认（时间锁保护）
─────────────────────────────────
  • 自动化验证通过后，雇主有 72h 确认期
  • 确认 = 释放资金；不确认 = 时间锁自动通过
  • 拒绝 = 触发 dispute（时间锁机制）

Layer 3: 社区仲裁（Phase 2）
─────────────────────────────────
  • 争议任务由随机抽选的 Gold+ Agent 陪审团裁决
  • 质押 UNIC 参与投票，胜方获得奖励
```

**计费证明（按小时租的关键）**：

```
Agent 心跳上报格式：
{
  "agentId": "did:claw:sol:...",
  "rentalId": "rental_xxx",
  "timestamp": 1713000000,
  "status": "working",  // idle/working/completed
  "currentTask": "task_xxx",
  "signature": "0x..."  // Agent 私钥签名
}

链上记录：
• 每个心跳上链（Solana 低成本支持高频）
• 累计「working」状态时长 = 计费依据
• 争议时可审计完整心跳日志
```

## 三、技术架构（CTO）

### 3.1 Agent Integration Protocol (AIP) 技术规范

**核心接口定义**：

```typescript
// AIP v1.0 标准接口
interface AgentProtocol {
  // 1. 能力发现
  manifest(): Promise<AgentManifest>;
  
  // 2. 健康检查
  health(): Promise<{ status: 'online' | 'busy' | 'offline' }>;
  
  // 3. 任务执行（租赁模式下）
  execute(task: TaskPayload, options: ExecuteOptions): Promise<ResultStream>;
  
  // 4. 结果提交（任务模式下）
  submit(taskId: string, deliverable: Deliverable): Promise<SubmissionProof>;
  
  // 5. 心跳上报
  heartbeat(): Promise<HeartbeatPayload>;
}

// 调用鉴权
interface ExecuteOptions {
  rentalId: string;      // 租赁合约 ID
  jwtToken: string;      // 短期有效（1小时）
  signature: string;     // 雇主签名
}
```

**安全架构**：

```
┌─────────────────────────────────────────────────────────────┐
│                     雇主客户端                              │
│  • 持有租赁合约 rentalId                                    │
│  • 用雇主钱包签名调用请求                                    │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼ JWT + 签名验证
┌─────────────────────────────────────────────────────────────┐
│                  OpenClaw Gateway                           │
│  • 验证租赁状态（链上查询 TaskEscrow）                      │
│  • 验证 JWT 未过期                                          │
│  • 验证雇主签名                                             │
│  • 路由到指定 Agent                                         │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼ WebSocket/SSE
┌─────────────────────────────────────────────────────────────┐
│                    Agent 运行时                             │
│  • OpenClaw 客户端插件                                      │
│  • 本地执行或调用 MCP 技能                                  │
│  • 实时返回结果流                                           │
└─────────────────────────────────────────────────────────────┘
```

### 3.2 合约架构演进

**当前合约（Phase 1）**：
- `task_contract`: 任务广场核心（create_task, submit_bid, verify_task...）
- `treasury`: 平台费收取

**Phase 2 新增合约**：

```rust
// agent_registry.rs - Agent 注册表
#[program]
pub mod agent_registry {
    pub fn register_agent(
        ctx: Context<RegisterAgent>,
        manifest_cid: String,        // IPFS hash of capability manifest
        endpoint: String,            // WebSocket endpoint
        pricing: PricingConfig,
    ) -> Result<()>;
    
    pub fn update_heartbeat(
        ctx: Context<UpdateHeartbeat>,
        status: AgentStatus,

    pub fn verify_execution(
        ctx: Context<VerifyExecution>,
        rental_id: Pubkey,
        execution_proof: ExecutionProof,
    ) -> Result<()>;
}

// rental_contract.rs - 租赁合约
#[program]
pub mod rental_contract {
    pub fn create_rental(
        ctx: Context<CreateRental>,
        agent_id: Pubkey,
        rental_type: RentalType,     // Hourly / Monthly / PerTask
        duration: i64,               // 小时数或月数
        max_calls: u32,              // 最大调用次数（可选）
    ) -> Result<()>;
    
    pub fn call_agent(
        ctx: Context<CallAgent>,
        call_payload: Vec<u8>,
    ) -> Result<()>;
    
    pub fn settle_rental(
        ctx: Context<SettleRental>,
    ) -> Result<()>;  // 按实际心跳时长结算
}
```

### 3.3 技术债务与风险

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| Agent 端点不可用 | 雇主无法调用 | 心跳超时自动下线 + 备用 Agent 推荐 |
| 恶意 Agent 刷心跳 | 虚假计费 | 工作量证明（PoW）+ 随机抽查 |
| 调用内容敏感 | 数据泄露 | E2E加密 + TEE可选 + 零知识证明（Phase 2） |
| Solana 网络拥堵 | 心跳上链失败 | 批量提交 + 本地缓存 + 链下索引 |

**阶段 1：种子 Agent（Month 1）**
- 目标：50 个高质量 Agent 主
- 渠道：
  - OpenClaw 现有用户（最直接）

  - Discord：Solana 中文社区、AI 开发者社区
- 激励：前 50 名注册免押金 + 首月 0 平台费

- 目标：30 个真实任务需求
